# 🔐 PageCraft Pro — 접근 제어 링크 시스템

## 개요

어드민이 URL을 생성하여 **특정 횟수**, **특정 기간** 동안만 접근 가능하게 제어하는 시스템입니다.
링크만 있으면 바로 콘텐츠에 접근할 수 있으며, 별도의 이메일 인증은 없습니다.

---

## 접근 제어 2요소

| 제어 항목 | 설명 | 예시 |
|-----------|------|------|
| **횟수 제한** | 총 방문 가능 횟수 설정 (0 = 무제한) | 최대 10회 |
| **기간 제한** | 만료 일시 설정 (null = 무기한) | 2026-04-30 까지 |

---

## 배포 방법

### 1. Vercel 환경변수 설정

```
ADMIN_PASSWORD=your_secure_password_here
GEMINI_API_KEY=기존_키_유지
```

### 2. (선택) Vercel KV 연결

영구 데이터 저장이 필요하면 Vercel KV를 연결합니다.

> ⚠️ KV를 연결하지 않으면 메모리 스토어가 사용됩니다.
> 서버리스 함수 cold start 시 데이터가 초기화될 수 있습니다.
> **프로덕션 환경에서는 반드시 Vercel KV를 연결하세요.**

### 3. 배포

```bash
vercel --prod
```

## 사용법

### 어드민: 링크 생성

1. `https://your-domain.vercel.app/admin` 접속
2. `ADMIN_PASSWORD`로 로그인
3. "새 링크 생성" 탭:
   - **제목** 입력
   - **최대 접근 횟수** 설정 (0 = 무제한)
   - **만료 일시** 설정 (비워두면 무기한)
   - **콘텐츠** 선택 (HTML / 외부 URL 리디렉트 / 이미지)
4. 생성된 URL을 대상자에게 공유

### 사용자: 콘텐츠 접근

1. 공유받은 URL(`/view?token=xxx`) 접속
2. 자동으로 검증 → 통과 시 바로 콘텐츠 표시
3. 실패 시 사유별 에러 메시지 표시:
   - ⏰ 기간 만료
   - 📊 횟수 초과
   - ⏸️ 링크 비활성화

### 어드민: 링크 관리

- **활성/비활성 토글**: 즉시 접근 차단/허용
- **방문 기록 조회**: 언제 접속했는지, IP 확인
- **방문 초기화**: 횟수 카운터 리셋
- **삭제**: 링크 영구 삭제

## API 엔드포인트

| 메서드 | 경로 | 설명 | 인증 |
|--------|------|------|------|
| POST | `/api/links/create` | 새 링크 생성 | Admin |
| GET | `/api/links/list` | 전체 링크 목록 | Admin |
| POST | `/api/links/manage` | 링크 수정/삭제/토글 | Admin |
| POST | `/api/links/verify` | 접근 검증 (사용자용) | 없음 |

## 파일 구조 (추가분)

```
pagecraft-final/
├── admin.html              ← 어드민 대시보드
├── view.html               ← 공유 콘텐츠 뷰어 (자동 검증)
├── api/
│   └── links/
│       ├── _store.js       ← 스토리지 추상화
│       ├── create.js       ← 링크 생성 API
│       ├── list.js         ← 링크 목록 API
│       ├── manage.js       ← 링크 관리 API
│       └── verify.js       ← 접근 검증 API
└── vercel.json             ← 라우팅 업데이트됨
```

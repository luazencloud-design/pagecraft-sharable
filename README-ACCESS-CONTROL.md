# 🔐 PageCraft Pro — 접근 제어 링크 시스템

## 개요

어드민이 URL을 생성하여 **특정 횟수**, **특정 기간**, **특정 사용자(이메일 기반)**에게만 콘텐츠 접근을 허용하는 시스템입니다.

---

## 아키텍처

```
[어드민 대시보드]          [공유 뷰어 페이지]
   /admin                    /view?token=xxx
      │                           │
      ▼                           ▼
┌──────────────┐         ┌──────────────┐
│ /api/links/  │         │ /api/links/  │
│  create.js   │         │  verify.js   │
│  list.js     │         └──────┬───────┘
│  manage.js   │                │
└──────┬───────┘                │
       │    ┌───────────────┐   │
       └───►│  스토리지 (KV) │◄──┘
            └───────────────┘
```

## 접근 제어 3요소

| 제어 항목 | 설명 | 예시 |
|-----------|------|------|
| **횟수 제한** | 총 방문 가능 횟수 설정 (0 = 무제한) | 최대 10회 |
| **기간 제한** | 만료 일시 설정 (null = 무기한) | 2026-04-30 까지 |
| **사용자 제한** | 허용된 이메일 목록만 접근 가능 | user@corp.com 등 |

## 배포 방법

### 1. Vercel 환경변수 설정

Vercel 대시보드 → Settings → Environment Variables에서 추가:

```
ADMIN_PASSWORD=your_secure_password_here
GEMINI_API_KEY=기존_키_유지
```

### 2. (선택) Vercel KV 연결

영구 데이터 저장이 필요하면 Vercel KV를 연결합니다:

1. Vercel 대시보드 → Storage → Create → KV
2. 프로젝트에 연결하면 자동으로 환경변수가 설정됩니다:
   - `KV_REST_API_URL`
   - `KV_REST_API_TOKEN`

> ⚠️ KV를 연결하지 않으면 메모리 스토어(fallback)가 사용됩니다.
> 이 경우 서버리스 함수가 cold start 되면 데이터가 초기화됩니다.
> **프로덕션 환경에서는 반드시 Vercel KV를 연결하세요.**

### 3. 배포

```bash
vercel --prod
```

## 사용법

### 어드민: 링크 생성

1. `https://your-domain.vercel.app/admin` 접속
2. `ADMIN_PASSWORD`로 로그인
3. "새 링크 생성" 탭에서:
   - **제목** 입력 (예: "2026 S/S 미리보기")
   - **최대 접근 횟수** 설정 (0 = 무제한)
   - **만료 일시** 설정 (비워두면 무기한)
   - **허용 이메일** 입력 (줄바꿈 또는 쉼표로 구분)
   - **콘텐츠** 선택:
     - HTML: 직접 HTML 작성
     - 외부 URL 리디렉트: 인증 후 다른 페이지로 이동
     - 이미지 URL: 이미지를 직접 표시
4. "접근 제어 링크 생성" 클릭
5. 생성된 URL을 허용된 사용자에게 공유

### 사용자: 콘텐츠 접근

1. 공유받은 URL(`/view?token=xxx`) 접속
2. 자신의 이메일 입력
3. 검증 통과 시 콘텐츠 표시
4. 검증 실패 시 사유별 에러 메시지 표시:
   - 🚫 이메일 미등록
   - ⏰ 기간 만료
   - 📊 횟수 초과
   - ⏸️ 링크 비활성화

### 어드민: 링크 관리

링크 목록에서 각 링크에 대해:
- **활성/비활성 토글**: 즉시 접근 차단/허용
- **방문 기록 조회**: 누가 언제 접속했는지 확인
- **방문 초기화**: 횟수 카운터 리셋
- **삭제**: 링크 영구 삭제

## API 엔드포인트

| 메서드 | 경로 | 설명 | 인증 |
|--------|------|------|------|
| POST | `/api/links/create` | 새 링크 생성 | Admin |
| GET | `/api/links/list` | 전체 링크 목록 | Admin |
| POST | `/api/links/manage` | 링크 수정/삭제/토글 | Admin |
| POST | `/api/links/verify` | 접근 검증 (사용자용) | 없음 |
